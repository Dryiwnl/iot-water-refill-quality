# TS-300B Turbidity Sensor

The TS-300B was implemented in the project to measure the turbidity of the water. The higher the turbidity, the greater the concentration of suspended particles, such as soil, mud, organic matter, or other contaminants. The sensor returns a numerical value corresponding to the water turbidity, typically expressed in **NTU (Nephelometric Turbidity Units)**.

![Turbidity Scale](/docs/Hardware/Sensores/img/turbidez.png)

---

# How It Works

The module has three pins:

* **VCC:** Power supply (5 V)
* **GND:** Ground
* **OUT:** Analog output

The sensor consists of two ends: a light emitter and a light receiver. The emitter sends a beam of light toward the receiver. As the amount of suspended particles in the water increases, part of the light is absorbed or scattered, reducing the intensity of the light reaching the receiver.

The lower the received light intensity, the higher the measured turbidity.

![Sensor Operation](/docs/Hardware/Sensores/img/img2.png)

---

As a result, the sensor outputs a voltage proportional to the amount of light received.

Since the module is configured in **analog mode** in this project, its output is read using the ESP32 Analog-to-Digital Converter (ADC).

The ESP32 ADC has a **12-bit resolution**, meaning:

```math
2^{12}=4096
```

Therefore, an ADC reading can range from **0 to 4095**.

To convert the ADC reading into voltage, the following equation is used:

```math
Voltage = ADC \times \frac{3.3}{4095}
```

Since the ESP32 ADC provides better resolution at lower voltages, configuring the ADC attenuation is recommended to extend its measurement range.

# Test Code

```cpp
#include <Arduino.h>

// Pin definition
#define TURB_PIN 32

// Voltage measured in clean water, used as the reference (0 NTU)
float tensao_limpa = 2.760;

// Gain used to convert the voltage drop into turbidity (NTU)
float ganho_turbidez = 1000.0;

void setup() {
  Serial.begin(115200);
  delay(1000);

  pinMode(TURB_PIN, INPUT);

  // 12-bit ADC resolution
  analogReadResolution(12);

  // ADC attenuation for a wider voltage measurement range
  analogSetPinAttenuation(TURB_PIN, ADC_11db);
}

void loop() {

  int adc = analogRead(TURB_PIN);

  float tensao = adc * (3.3 / 4095.0);

  float ntu = (tensao_limpa - tensao) * ganho_turbidez;

  if (ntu < 0)
    ntu = 0;

  Serial.print("Voltage: ");
  Serial.print(tensao, 3);

  Serial.print(" V | Turbidity: ");
  Serial.print(ntu, 1);

  Serial.println(" NTU");

  delay(500);
}
```

# Analog/Digital Configuration

The module provides three solder pads to select its operating mode:

* **A:** Analog
* **B:** Bridge
* **D:** Digital

Depending on which pads are soldered together, the sensor operates in a different mode.

| PAD | Solder Configuration    | Operation                                                              |
| --- | ----------------------- | ---------------------------------------------------------------------- |
| A   | A soldered to B         | The module operates in analog output mode.                             |
| B   | $\bar{A}$ and $\bar{D}$ | The module is not configured correctly, resulting in invalid readings. |
| D   | D soldered to B         | The module operates in digital output mode.                            |

![Pad Configuration](/docs/Hardware/Sensores/img/img1.png)

Most vendors ship the module already configured for **analog operation**. However, this configuration can be modified according to the application requirements.

In this project, the **A soldered to B** configuration was used, enabling analog measurements.
